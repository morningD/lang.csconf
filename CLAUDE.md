# lang.csconf - Project Status & Notes

## Project Overview
Visualizing linguistic diversity of first authors across CCF-rated CS conferences (2010–2026).
Pipeline: conferences_base.json + CCF PDFs → DBLP SPARQL crawl → name classification → JSON stats → Vue 3 website.

## CRITICAL: Safe Re-Crawl Procedure
**NEVER run `--step 2 --force` without immediately following with `--step 2c --force`.**

Step 2 `--force` overwrites ALL `data/raw/authors/` files with fresh SPARQL data. Step 2c gap-filled files (auto or manual) that have `_source != "sparql"` will survive ONLY if SPARQL returns 0 papers for that conference-year (step2 skips empty results). But step 2c auto-filled gaps where SPARQL returns 1-19 papers WILL be overwritten with the sparse SPARQL data.

**Safe full re-crawl sequence:**
```bash
python -m pipeline.run_all --step 2 --force    # Re-crawl SPARQL
python -m pipeline.run_all --step 2c --force    # Re-fill all gaps (MUST run after step 2 --force)
python -m pipeline.run_all --step 2d --force    # Re-fill OpenReview gaps
python -m pipeline.run_all --step 3 --force    # Re-classify
python -m pipeline.run_all --step 4            # Regenerate stats
rsync -av --delete data/stats/ website/public/data/stats/
```

**Manual gap fills are safe** if SPARQL returns 0 for that conference-year (e.g., ISMB journal papers, HiPEAC API data, FSE-CRYPTO IACR data, ECAI cross-filed, MLSys website scrape) — step2 doesn't overwrite files with empty results. But always verify after a full re-crawl.

**Before any destructive operation, always ask the user for confirmation.** This includes `--force` flags on any pipeline step.

- **CCF rank history**: 387 conferences across 6 editions (2011 partial, 2012, 2015, 2019, 2022, 2026)

**Institution normalization**: `_normalize_institution()` in step4 handles all institution name normalization → `(canonical_name, country_code)`. Returns a tuple, not just a string.
- **Normalization pipeline** (in order):
  1. Strip trailing parenthetical content: `re.sub(r'\s*\([^)]+\)\s*$', '', name)` — handles `(MIT)`, `(Guangzhou)`, `(Shenzhen)`, `(KAIST)` etc.
  2. Exact lookup in `_INST_MAP` (~250 entries): raw name → (canonical, country)
  3. Case-insensitive lookup fallback
  4. OpenReview duplicate cleanup: "X, X" → "X"
  5. Prefix stripping: "Department/School/Faculty of ..." → rest
  6. Comma-separated segment matching: find the segment containing "university"/"institute" and match against map
  7. Address suffix stripping: remove ", City, ST, USA" / ", Country" patterns
- **Key design decisions & pitfalls:**
  - **Parenthetical stripping is generic, not per-variant**: Don't add individual map entries for "(MIT)", "(UCL)", "(KAUST)" — one regex rule handles all. Only add map entries when the base name itself differs (e.g., "UC Berkeley" vs "University of California, Berkeley").
  - **Campuses are merged, not split**: "HKUST (Guangzhou)" → "HKUST", "HIT (Shenzhen)" → "Harbin Institute of Technology". Different campuses are in the same country and we don't distinguish them.
  - **papercopilot long addresses**: Many entries have full department + address (e.g., "Mechanical Engineering, MIT, Cambridge, MA, USA", "Computer Science, University of Freiburg, Germany"). The comma-segment matching + address stripping handles most of these. Remaining edge cases go in `_INST_MAP`.
  - **Country codes come from the map, not raw data**: papercopilot has no country field, OpenReview is often empty. The `_INST_MAP` is the authoritative source for country codes. Raw `institution_country` is only used as fallback.
  - **Only top-10 need normalization**: The mapping focuses on institutions appearing in any conference's top-10 (~110 unique names). New institutions entering top-10 in the future will need new map entries.
  - **`_INST_MAP` vs `INST_ABBREV`**: `_INST_MAP` (in step4) maps raw → canonical names for aggregation. `INST_ABBREV` (in ConferenceView.vue) maps canonical → display abbreviations. Both dicts must stay in sync when adding new institutions.

## Key Technical Details

### Data Model
- **Raw author format**: `{name, title, year, ordinal}` — flat, one entry per author per paper
- **All authors stored** with `ordinal` field (1=first, 2=second, etc.); step4 filters to `ordinal==1`
- **Languages**: Chinese, English, Korean, Japanese, German, French, Indian, Spanish, Italian, Russian, Portuguese, Persian, Arabic, Vietnamese, Turkish, Dutch, Other
- **Name classifier**: fastText (fast-langdetect lid.176.bin) + rule-based surname ensemble. Rule-based wins at ≥0.6 confidence, fastText catches rest. Dependency: `pip install fast-langdetect`
- **Conference notes**: Two levels — (1) `note` field in `conferences_base.json` for conference-level notes; (2) `pipeline/conference_year_notes.json` for per-(conference, year) provenance, metadata, and anomaly tracking. Both propagated through step4 into stats JSON.
- **Year notes fields**: `date`, `location`, `homepage`, `paper_source`, `paper_source_url`, `accept_rate_source`, `status` (ok/noise/gap/virtual/partial/uninvestigated), `note`

### DBLP Integration
- **SPARQL endpoint**: `https://sparql.dblp.org/sparql`, batch size 5, 0.2s rate limit
- **Data filtering (unified at entry point)**: All filtering and deduplication happens when data enters the system, NOT during stats generation. Step4 does pure aggregation with only a lightweight venue-year correction.
  - **Workshop filter**: `_EXCLUDED_VENUE_RE` in `dblp_sparql.py` filters `Workshop|Companion|Tutorial|@` from SPARQL results
  - **Proceedings volume filter**: `is_proceedings_volume()` in `pipeline/utils/filters.py`, applied in both `dblp_sparql.py` (step2) and `step2c_fill_gaps.py`. Catches proceedings editor entries: titles ending with "Proceedings", "Revised Selected Papers", "Lecture Notes in Computer Science", "Proceedings of the Nth...", "Frontmatter", "Letter from the Chair", etc. ~3,200 such entries cleaned from source data (2026-03-11).
  - **Publication year split merging**: `merge_publication_year_splits()` in step2 merges files where SPARQL yearOfPublication differs from the real conference year (using venue data). Runs automatically after step2 and step2c. Also deletes stale classified files so step3 re-classifies the merged data.
- **Publication year ≠ conference year**: DBLP `yearOfPublication` can lag the real conference year by 1 (proceedings published the year after). Two mechanisms fix this:
  1. **URI year extraction (in SPARQL query)**: `dblp_sparql.py` extracts the real year from paper URI keys (e.g., `conf/iconip/FooBar23` → suffix `23` → 2023). Always trusted when in [YEAR_FLOOR, year_ceiling()] range. Proceedings volume keys like `2023-10` (Part 10) are filtered by `_PROCEEDINGS_KEY_RE`. Out-of-range years (e.g., 2089) fall back to `yearOfPublication`. **Validated 2026-03-11**: comparison with HTML proceedings pages shows URI year matches within 1-2 papers for all tested conferences (ICONIP, PRCV, SECURECOMM, PDCAT).
  2. **Post-crawl merging (`merge_publication_year_splits()` in step2)**: Two passes — (a) Non-venue-year merge: when year Y is NOT a venue year but Y-1 IS, merge Y into Y-1; (b) Cross-year dedup: when consecutive year files share ≥50% paper titles (min 5 papers), remove overlapping papers from the later-year file. Runs after both step2 and step2c. Also deletes stale classified files.
  - **Caveat**: To capture papers for conference year Y, SPARQL must query `yearOfPublication` in {Y, Y+1} since proceedings may be published a year late. `year_ceiling()` in `pipeline/utils/years.py` auto-extends to `current_year + 1`.
  - **SIGMETRICS-type journal papers**: Some conferences (SIGMETRICS/POMACS) publish papers in journals that appear in the SPARQL stream but not on proceedings HTML pages. SPARQL correctly includes these (~25 extra papers/year for SIGMETRICS). This is desired behavior.
- **DBLP HTML caveat**: Do NOT use custom User-Agent headers when fetching DBLP HTML. DBLP serves truncated lazy-loaded HTML to browser-like UAs. Default `python-requests` UA gets full content.
- **Venue data**: Scraped from DBLP HTML index pages (`<h2>` headings like "EuroSys 2024: Athens, Greece"). Handles virtual events, city-states, slash-separated dual venues.
  - **Political sensitivity mapping**: Taiwan/Hong Kong/Macau automatically converted to "Taiwan, CN"/"Hong Kong, CN"/"Macau, CN" in `step2b_crawl_venues.py` via `REGION_TO_COUNTRY` dict. Applied during crawl and can be batch-updated via `scripts/update_venue_regions.py`.
- **Year range filter**: Step4 skips papers with year < 2010 or year > year_ceiling() to filter DBLP data errors.

### DBLP Key Management
**CRITICAL: Always verify DBLP keys on https://dblp.org/db/conf/{key}/ before adding/changing.** Keys are often unintuitive (e.g., `pact` = PaCT not PACT, `fg` = Formal Grammar not Face & Gesture).

All DBLP keys are stored in `pipeline/conferences_base.json`. Do NOT edit `data/raw/conferences.json` — step1 regenerates it.

## Gap Filling — Data Source Hierarchy

### 1. DBLP SPARQL (step2) — Primary
Fastest, most complete for well-indexed conferences. Batch queries with workshop filtering.

### 2. DBLP Search API (step2c) — Auto gap fill
Detects gaps: SPARQL < 20 papers AND venue data exists AND ±3-year median ≥ 30.
Fills via `venue:{keyword} year:{year}` queries, filtered to proceedings page keys.

**Detection limitations:**
- Circular median: When ALL conference years are gaps (0 papers), median=0 → no detection (affected ACCV)
- Small conferences: Median below MIN_EXPECTED=30 → missed (affected SOUPS pre-2017)
- Partial gaps: 20-50% of expected papers above threshold → missed (affected IEEEVIS 2024)

### 3. DBLP HTML Scraping (step2c fallback)
When Search API returns 500 errors, parse `<li class="entry">` blocks from proceedings pages.

**Format warning**: `fetch_papers_from_html()` returns nested `{title, authors: [{name, ordinal}]}`. Must flatten to `{name, title, year, ordinal}` before saving, else step3 crashes.

### 4. OpenReview (step2d) — Auto supplementary source
OpenReview (openreview.net) hosts paper data for major AI/ML conferences, often available months before DBLP indexes the proceedings. Step2d automatically checks registered venues and fills missing conference-years.

**Registered conferences**: NEURIPS, ICLR, ICML, AISTATS, CoRL, UAI, COLM (configured in `pipeline/utils/openreview.py` VENUE_REGISTRY).

**Behavior**:
- Only fills conference-years where no raw author file exists (incremental)
- `--force` re-crawls only OpenReview-sourced files (`_source: "openreview"`), never overwrites SPARQL data
- Also extracts acceptance rates when submission count is available (submitted > accepted)
- Cleaned up stale classified files for re-classification by step3

**Adding a new OpenReview conference**: Add entry to `VENUE_REGISTRY` in `pipeline/utils/openreview.py` with the venue-id pattern (e.g., `"CONF": "org.org/CONF/{year}/Conference"`).

**SSL issues**: OpenReview API (`api2.openreview.net`) has intermittent SSL errors (`SSLEOFError: UNEXPECTED_EOF_WHILE_READING`). Workaround: `s.verify = False` + `urllib3.disable_warnings()` in `_session()`. Some conference-years may fail with `ConnectionError` — retry later.

### 4b. OpenReview Affiliation Enrichment (step 2e, automatic)
For the 7 OpenReview conferences, step 2e fetches first-author profiles to extract institution data:
- Profile API: `GET /profiles?id=~Profile_Name` → `content.history[0].institution.{name, country}`
- Rate limited to ~3 RPS (`time.sleep(0.3)` per profile fetch)
- Persistent profile cache at `data/raw/affiliations/_profile_cache.json` avoids re-fetching known profiles
- Coverage: ~92-93% for NeurIPS/ICLR/ICML papers (profile IDs available for most authors)
- `_normalize_institution()` handles OpenReview artifacts like repeated names in institution field
For conferences that publish in journals (ISMB/ECCB → Bioinformatics Supplements).

**ISMB odd years (2011-2023)**: Joint ISMB/ECCB papers published in Bioinformatics journal supplements, not `conf/ismb/`. Recovered 509 papers by scraping DBLP journal volume pages:
- URL pattern: `https://dblp.org/db/journals/bioinformatics/bioinformaticsNN.html#anchor`
- Find supplement section by anchor ID → extract `<li class="entry article">` entries
- Authors in `<cite>` block: `itemprop="name" title="Author Name"`
- Skip non-paper entries (proceedings headers, awards, editorials)
- Anchor mappings: 2023→`nrSupplement-1`, 2021→`nrSupplement1`, 2019/2017→`nr14`, 2015→`nr12`, 2013/2011→`nr13`

### 6. Conference Website Scraping (manual)
For conferences with journal-first model and no DBLP conf entries.

**HiPEAC (2015-2025)**: Papers published in ACM TACO journal, not in `conf/hipeac/`. Recovered 294 papers:
- HiPEAC API: `https://www.hipeac.net/api/v3/events/{event_id}/activities/` → Paper Track sessions
- Session detail: `/api/v3/events/activities/{id}/` → description field has paper titles + authors
- Authors completed via CrossRef DOI lookup (`https://api.crossref.org/works/{doi}`) — more reliable than DBLP Search API
- DOI extraction from descriptions: 2019 uses `citation.cfm?id=XXXXXXX`, 2020-2024 have `10.1145/...`, 2025 has TACO-IDs (search by title)

### 7. ACM TECS Journal Special Issues (ESWEEK conferences)
CASES, CODES+ISSS, and EMSOFT adopted a journal-first model from ~2017. Full papers publish in ACM TECS Vol N, No 5s (odd years: 2019, 2021, 2023, 2025). DBLP `conf/` entries only have WiP/Demo papers. Even years (2018, 2020, 2022, 2024) have no TECS 5s special issue (ACM DL returns 404); low paper counts for even years are structural.

**Recovery procedure:**
1. Download BibTeX from ACM DL section pages: `https://dl.acm.org/toc/tecs/{year}/{year-2001}/5s?tocHeading=heading{1,2,3}`
   - heading1 = CASES, heading2 = CODES+ISSS, heading3 = EMSOFT
2. Parse BibTeX, fix LaTeX-encoded names (ö, é, ç, etc.)
3. Merge with existing SPARQL data (usually zero overlap: SPARQL has WiP, TECS has full papers)
4. Mark `_source: "journal_scrape+sparql"`

**Recovered:** 2019 (+39 papers), 2021 (+18), 2023 (+59), 2025 (+72). Total ~190 papers.

**Note:** ACM DL main page lazy-loads; use `?tocHeading=headingN` URLs for individual sections. Cloudflare may block — use Playwright or download BibTeX manually.

### 8. Generic Website Scraping (manual, last resort)
For non-archival or completely unindexed conferences. Example: SysML/MLSys 2018 scraped from mlsys.org (116 papers). Mark with `_source: "website_scrape"`.

## Data Filtering & Validation Architecture
**Filtering (at data entry point, NOT during stats generation):**
- `dblp_sparql.py` (step2): workshop filter (`_EXCLUDED_VENUE_RE`) + workshop abbreviation filter (`_find_workshop_abbreviation_venues`, detects `{Dominant}W`/`{Dominant}-W` patterns like ICCVW, ICSTW, DSN-W) + proceedings volume filter (`is_proceedings_volume` from `utils/filters.py`) + skips empty results + skips out-of-range years (<2010 or >2026)
- `step2c_fill_gaps.py`: proceedings volume filter (same shared `utils/filters.py`)
- `step2d_crawl_openreview.py`: fills missing years from OpenReview, skips existing SPARQL data, also extracts acceptance rates
- `step4_generate_stats.py`: pure aggregation, only `ordinal==1` first-author selection + year range filter (auto: 2010 to current_year+1) + publication-year correction (SPARQL data only) + `skip` flag from `conference_year_notes.json` (skips joint conference years already counted elsewhere)

**Anomaly detection (step4, runs every time before stats generation):**
- **No venue data**: Flags conference-years with papers but no venue data (conference hasn't happened yet)
- **Sharp drop**: Paper count < 40% of recent average (excluding COVID 2020-2021)
- **Spike**: Paper count > 2x recent max (or 3x if prior years show consistent growth trend) AND > 100 more papers than recent max
- Anomalies documented in `conference_year_notes.json` are suppressed (shown as known count only). Only NEW anomalies trigger full warnings. Conference-years with `skip: true` are excluded from detection.

**Filename validation (prevents stale/duplicate files from polluting stats):**
- `step3_classify_names.py` and `step4_generate_stats.py` validate filenames with `_VALID_AUTHOR_FILENAME` regex: `^[A-Za-z0-9+&.\-]+_\d{4}\.json$`
- Rejects macOS Finder " 2" duplicates, files with spaces, and other anomalies
- If invalid files are found, step4 prints a warning

Workshop filtering reduced total from ~957k to ~884k. Workshop abbreviation filter removed ~1,643 additional entries (2026-03-12). Proceedings volume cleanup removed ~3,200 entries.

## Academic Database Verification via Playwright

When DBLP SPARQL data is missing or suspicious, use the `academic-search` skill (Playwright + Edge + ECNU proxy) to verify directly on publisher databases.

### Quick Verification Methods
- **ACM conferences**: `https://dl.acm.org/conference/{id}/proceedings` — lists all years with proceedings, fast and reliable through proxy
- **IEEE conferences**: `https://ieeexplore.ieee.org/search/searchresult.jsp?queryText="{conf name}"` — JS-heavy, use `browser_wait_for` before reading
- **DBLP bulk queries**: Use SPARQL API directly (`requests.post` to `https://sparql.dblp.org/sparql`) — much faster than browser. Must include both `PREFIX dblp:` and `PREFIX xsd:` declarations.

### Key Findings
- ACM DL uses event links (`/do/10.1145/event-{id}/abs/`) for journal-model conferences vs DOI links (`/doi/proceedings/...`) for traditional proceedings
- DBLP pages lazy-load content in browser — often appear empty; SPARQL API is the reliable alternative
- `browser_evaluate` is essential for extracting structured data from heavy pages (paper counts, titles, DOIs) instead of parsing enormous snapshots

### Journal-First Conference Detection
Some conferences switched to journal-first model (papers in journals, not `conf/` on DBLP):
- I3D 2021+: papers in PACMCGIT (Proceedings of ACM on Computer Graphics and Interactive Techniques)
- ESWEEK (CASES/CODES+ISSS/EMSOFT) odd years: papers in ACM TECS
- Detection: ACM DL shows proceedings but DBLP SPARQL returns 0; ACM event page lists `research-article` entries
